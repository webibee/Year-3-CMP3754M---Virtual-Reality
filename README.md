# Urban Street Scene VR – Pedestrian Interaction with Automated Vehicles
- Course: CMP3754M – Virtual and Augmented Reality
- Institution: University of Lincoln
- Grade Received: 80%
- Platform: Meta Quest 2 (Oculus)
- Engine: Unity 2020.3.40f1
- Hardware: Head-Mounted Display (HMD) with hand controllers

# Overview
This project is an immersive virtual reality simulation of an urban street environment, designed to run on the Meta Quest 2 headset. The scene includes buildings, pedestrians, traffic systems, and a player-controllable car. The simulation is intended for research into pedestrian interaction with automated vehicles, with a focus on user experience (UX), presence, embodiment, and locomotion.

The project was developed through a series of VR workshops (Workshops 2-6) and includes challenge features such as:
- Cars stopping for pedestrians
- Traffic light systems
- Pedestrian pathfinding and animations
- Audio integration (engine sounds, ambient shop noise, bird sounds)
- Hand-tracking and controller-based interaction

# Key Features
Core Workshop Requirements:
- Urban street scene with buildings, pedestrians, and cars
- Pedestrian pathfinding and walking animations
- Traffic system with cars following waypoints
- Player-controlled car with driving mechanics
- Hand-tracked controllers with hand models in VR
- Head tracking for 6 Degrees of Freedom (6DoF) movement

Challenge Features Implemented:
- Cars stop for pedestrians crossing the street
- Doppler effect on car engine sounds
- Ambient audio (shop interiors, park birds)
- Unique pedestrian models (no two identical)
- Custom car spawn script

Extended Challenge (Partial / Suggested)
- Car acceleration/deceleration (discussed as improvement)
